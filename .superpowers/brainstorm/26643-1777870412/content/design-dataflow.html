Design — Section 2 of 4: Data Flow

How a lead moves through the system — from first search to provisioned paying client.

1
Search → Lead Created
Efton searches "dentists in Austin." Google Places returns results. A lead record is created in Supabase with business name, address, phone, rating, review count, hours, category. Status: New.
2
Profile Viewed → Claude Snapshot Generated
Efton clicks the lead. Claude analyzes the business data and writes a snapshot — pain points, missed follow-up opportunities, pitch angle. Status: Reviewed.
3
Demo Run → ElevenLabs Agent Speaks
Efton launches the live voice demo — in person, via video, or via a shareable link. The ElevenLabs agent uses the business data to have a real conversation. Status: Demoed.
4
Proposal Sent → ACE Takes Over
Branded proposal sent via Resend. ACE begins automated follow-up sequence — texts, emails, call booking nudges — until the prospect responds or books. Status: Proposal Sent.
5
Call Booked → Deal Closed
Prospect books a call. Efton (and Sabrina on pricing) closes. Client signs. Payment collected. Status: Closed. Client record created in Supabase.
6
Provisioning → Client Goes Live
Efton + Claude Code configure the client's Revenue Rescue setup — ElevenLabs agent tuned to their business, Supabase tables for their leads, n8n workflow activated. Client goes live within 48 hours. Status: Active.
7
Revenue Rescue Running → ROI Proof
Client's AI handles their inbound. Supabase tracks every lead handled, follow-up sent, call booked. Monthly report sent. Client sees the ROI. Renewal is a no-brainer. Status: Recurring.

Does this look right? Stages 1–5 are already built and working. Stages 6–7 are what Tier 1 adds to the pipeline. The CRM in Layer 1 tracks status across all 7 stages so you always know where every lead and every client sits.
