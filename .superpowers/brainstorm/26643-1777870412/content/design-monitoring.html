Design — Section 4 of 4: Error Handling & Monitoring

What happens when something breaks — and how you find out before the client does.

What Can Break
🔍Google Places API — quota hit or key expired. Lead search returns empty.
🤖ElevenLabs agent — voice demo fails to launch or disconnects mid-call.
📨Resend email — proposal fails to send. Client never gets the follow-up.
🗄️Supabase — write fails. Lead status not saved. Pipeline out of sync.
⚙️Revenue Rescue n8n — client's follow-up workflow stalls. Leads fall through.
How We Catch It
📱Telegram alerts via VERA — any API error or n8n failure pings you immediately.
📋Vercel logs — every API route error is captured. Review after any report of a problem.
🗄️Supabase dashboard — live table view to confirm data is writing correctly.
⚙️n8n execution log — every workflow run is logged. Failed runs are visible immediately.
👀30-day concierge window — for clients 1–5, Efton manually checks their workflow weekly.
When Something Breaks — The Protocol
Step 1 — Contain
Is the client affected right now? If yes, pause their workflow in n8n immediately. Send a manual message to cover the gap. Do not wait to diagnose first.
Step 2 — Diagnose
Check Vercel logs, n8n execution log, and Supabase table in that order. Identify which layer failed. Efton + Claude Code investigate and fix.
Step 3 — Document
Add the failure to the runbook. What broke, what caused it, what fixed it. This becomes the first entry in the automation monitoring spec.
What You Tell Clients

During the concierge period: "I personally monitor your system weekly and am available if anything needs attention." That is the service level commitment — not a 99.9% uptime guarantee. It is a human backstop, which is actually more reassuring to a small business owner than any SLA document.

Does this look right? No new monitoring infrastructure needed in Tier 1. VERA already pings you on failures, n8n already logs every execution, Vercel already captures every error. The concierge window and the runbook documentation are the quality layer on top.
