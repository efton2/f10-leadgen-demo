Design — Section 1 of 4: System Architecture

How the platform is structured. Does this look right before we go deeper?

🌐
Layer 1 — Frontend (Vercel + Next.js)
The tool you use daily. Lead search, business profiles, live demo launcher, proposal sender, pipeline/CRM view. Password-protected. F10 Strategy brand. Lives at app.f10strategy.com.
↕
⚙️
Layer 2 — API Routes (Next.js + Node)
Google Places search, Claude AI snapshots, ElevenLabs agent creation, Resend proposal emails, ACE follow-up triggers. Already built and working.
↕
🗄️
Layer 3 — Data (Supabase)
Lead pipeline table, client records, provisioning status, ACE follow-up log, Revenue Rescue metrics. Single source of truth for all pipeline data.
↕
🤖
Layer 4 — Delivery (n8n + ElevenLabs + Claude)
Revenue Rescue workflow. Configured per client by Efton + Claude Code. Handles the client's inbound leads, follow-up, and ROI reporting. Manual for clients 1–5, then automated.
What is already built ✓
  • Lead search (Google Places)
  • Business profile + Claude snapshot
  • Live AI voice demo (ElevenLabs)
  • Proposal email (Resend)
  • ACE follow-up agent
  • Revenue Rescue n8n workflow
What Tier 1 adds
  • Login / auth (password protection)
  • Lead pipeline view (CRM)
  • Lead status tracking (new → demo → proposal → closed)
  • Client records (who is provisioned, what stage)
  • Subdomain: app.f10strategy.com

Does this look right? Four layers — frontend tool, API routes, Supabase data, and delivery automation. Everything you already built lives in layers 1–3. Tier 1 adds auth and CRM to layer 1. Revenue Rescue lives in layer 4 and gets configured per client.
